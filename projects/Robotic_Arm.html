Really? A robot?

Yeah cool isn't it? It is a child's dream, so I thought about using my electronics knowledge to pull it off. In this project, I used an ESP32 to control the robot's movements remotely. The ESP32 is a microcontroller which is the brains of the operation, it handles bluetooth communication between the phone app (yeah made that too), as well as the robot itself. Effectively making it so you can control the hand from a distance. The robot was also published along with a research paper to an international organization called CIIT and presented at Hotel Sirius Strumica in 2024. You can read more about it below:

How it was made

Components

  • 1 x Sheet of Plywood: This readily available and inexpensive material provides a sturdy yet lightweight base for the robotic arm.

  • 1 x ESPRESSIF ESP32 Devmodule: The brain of the operation, translating smartphone app commands into precise movements for the servo motors.

  • 4 x Servo motors sg90: These compact units act as the arm's muscles, precisely rotating the joints based on commands.

  • 1 x Battery Holder (3 cell): Provides power for both the servos and the ESP32 using standard AA batteries.

  • 1 x Stable 5V Power Delivery: Ensures reliable servo operation using an Arduino Nano, a voltage step-up module, or a rechargeable Li-Ion battery (with charger).

  • (Optional) 1 x 3.7V Rechargeable Li-Ion Battery: As an alternative to a battery holder this could be used with a Li-Ion battery charger module for infinite re-use.

Considerations

Firstly I was thinking about which material to use to build the frame out of. 3D printing could work but I didn't have access to a printer as they're very expensive, and even if I found one the plastic would cost allot. So I decided to go with plywood, my dad had a few sheets lying around so I used that. It is a cheap material, easy to work with, very sturdy and most importantly cheap.

As for the electronics I was dead-set on an ESP32 because it contains much more memory than any Arduino which allows me to go wild with the bluetooth protocol and libraries. It also has a bluetooth module built-in. Isn't that amazing?

The servos were a no-brainer, I had a few lying around from a previous project so I used those. They're cheap, easy to use, but had some issues with the SG90's pulling more current than the battery could provide. Additionaly they sometimes couldn't handle the weight of the arm so I had to use some tricks up my sleeve. Making the screws which the arm pivots around more loose. Also I effectively made a lever system (pivots around the front legs) so it can push the arm up allot more easily. In the future I plan to add metal gear servos with allot more torque.

The battery holder I just had lying around, it would of-course be allot better to use a rechargable Li-Ion battery. However that would require upping the voltage to 5V from 3.7V with a step-up module. Also a potential upgrade in the near future.

This was all first tested on a breadboard with potentiometers until later transfering to a solderable prototyping board. Making the electronics allot more compact so you can put them inside the robot base. The robot Bluetooth feature was later added after testing.

The Gripper

Building this robot required an end-effector to manipulate objects, the initial testing was done on a piece of cardboard until later moving on to the plywood. Essentialy the left and right side of the servo horns move in opposite directions, which means if we connect a wire coming from either one and then move the pivot of each of the servo 'fingers' it will move them to the same point in the middle where they touch. Stronger servo means more grip, a good locking servo can also help with it.

Movement

In consideration for this, I put myself to work in cutting all the necessary plywood pieces as I knew I needed the structural integrity. The plywood sheet was cut into separate rectangular wooden strips using a machine saw. After I brought them home I could further cut them into smaller chunks with a hand saw. The more complicated pieces like the gripper fingers and the shoulder were cut a little more precisely. A few drilled holes later I could connect each of the pieces with screws and nuts to act as pivot points. The more the nuts are tightened the more the arm can hold its position. The first servo was attached to the base (which was glued together with hot glue) moved the robot with horizontal rotation the second was at the side providing vertical movement and the third at the front leg allowing the robot to go back and forth. Effectively this allowed the robot to move in all 3 dimensions.

Mobile app

I experimented with different ways of controlling the robot remotely, one of them was using a PS4 controller I borrowed. It wasn't particularly long-term and the libraries were very outdated, none seemed to work. So I decided to make my own app. I used MIT App Inventor, a block-based programming language which would speed up the process and allow me to focus on the bluetooth communication.

The APP sends two integer digits to the ESP32, the first is the button number and the second is the boolean state. So whether on or off, the Arduino code takes this as an input inside a switch-case and moves the servos accordingly.

A video is provided as a demonstration, just note that some of the screws were loose in that demonstration if you can notice some wobble. This is easily fixed.
